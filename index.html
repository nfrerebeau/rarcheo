Avant propos

Ce livre s’adresse aux personnes qui souhaitent analyser, visualiser et modéliser des données archéologiques à l’aide du langage R.1 Cet ouvrage vous permet d’acquérir les connaissances et les compétences nécessaires pour aborder un large éventail de problèmes liés aux données archéologiques.

Bien qu’il soit initialement conçu pour les étudiants du master mention Archéologie, sciences pour l’archéologie de l’Université Bordeaux Montaigne, ce livre s’adresse à tout étudiant, doctorant, post-doctorant ou archéologue professionnel souhaitant découvrir le langage R et s’y initier pour l’analyse de données archéologiques.

Ce livre s’adresse à des utilisateurs débutants, il ne demande aucun prérequis en programmation. Bien que des notions élémentaires soient rappelées au fil de l’ouvrage, ce livre n’est pas un manuel de statistiques. Le lecteur trouvera des suggestions de lecture complémentaires à la fin de l’introduction2.

Ce livre est en cours de rédaction.

Conventions d’écriture

Par la suite, on écrira foo() pour désigner une fonction, bar pour désigner les variables et les paramètres des fonctions et baz/ pour désigner les chemins. Les noms de packages sont indiqués en italique.

Les commentaires dans le code sont précédés par le signe #. Les blocs de code alternent l’entrée et la sortie. La sortie est commentée (précédée par #> pour la distinguer des commentaires standards) de sorte que vous pouvez facilement copier et coller les exemples dans R.

Reproductibilité

Ce livre a été écrit dans RStudio avec le package bookdown.5 Le site web est hébergé par GitHub Pages et automatiquement mis à jour après chaque commit par GitHub Actions à l’aide de pandoc version 2.14.2 (cette version a été compilée le 03 Mar 2022).

Les données utilisées dans les exemples sont issues des packages datasets6 et folio.7 Les exemples reposant sur la génération de nombre aléatoires sont reproductibles en utilisant set.seed(12345).

Cette version du livre a été réalisée avec R version 4.1.2 (2021-11-01), les packages utilisés sont résumés dans le tableau 0.1.
